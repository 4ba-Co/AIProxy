⚠️ Azure OpenAI 特别说明

Azure OpenAI服务使用特殊的URL格式：
https://ai-proxy.4ba.ai/azure-openai/{resource_name}/{deployment_name}/...

请将{resource_name}替换为您的Azure资源名称，{deployment_name}替换为您的部署名称。

💻接入示例

以下展示如何通过我们的代理服务访问OpenAI的Chat Completions API：

curl -X POST "https://ai-proxy.4ba.ai/openai/v1/chat/completions" \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_OPENAI_API_KEY" \ -d '{ "model": "gpt-4", "messages": [ { "role": "user", "content": "Hello, world!" } ], "max_tokens": 150, "temperature": 0.7 }'

🔑重要说明

⚠️ API密钥要求

您必须使用各AI服务提供商的官方一级API Key，需要自行在相应平台购买额度或注册账号。我们不提供任何二级API Key销售服务。

🔍 请求追踪与验证

由于我们仅做透明代理转发，您可以在各AI服务提供商的官方控制台或trace页面查看所有通过我们代理的请求记录，确保请求的完整性和透明度。

🛡️服务承诺

✅ 我们的承诺

  • 提供完全免费的代理服务
  • 仅做代理转发，绝不销售API密钥
  • 零缓存策略，不存储任何请求数据
  • 零日志策略，不跟踪用户请求行为
  • 保障数据传输的完整性和安全性
  • 确保与原始API的100%兼容性
